## Pourquoi utiliser Pawnote ?

<CardGrid stagger>
  <Card title="Entièrement typé" icon="rocket">
    Chaque requête et réponse de l'API est entièrement typé dans la librairie,
    rendant votre code sûr.
  </Card>
  <Card title="Support pour la version 2023" icon="approve-check">
    Pawnote supporte la dernière version de PRONOTE, et devrait aussi supporter la version 2022 (et celles d'avant)
    mais quelques propriétés peuvent être `undefined`.
  </Card>
  <Card title="Adapté pour tout usage" icon="star">
    Vous pouvez customiser la fonction utilisée dans librairie pour effectuer les requêtes HTTP,
    ainsi vous pouvez utiliser n'importe quelle API HTTP que vous voulez.
    Par défaut, Pawnote utilise l'API `fetch` donc Node.js, React Native et plus sont supportés sans configuration !
  </Card>
  <Card title="Même utilisation que l'interface" icon="puzzle">
    Pawnote essaye d'avoir le plus possible une utilisation qui se rapproche de l'interface officielle, ainsi la librairie peut vous fournir le plus de contexte possible en fonction de ce que vous demandez.
  </Card>
</CardGrid>
